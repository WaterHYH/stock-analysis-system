# 股票列表页 UI 全面优化 — 设计文档

**日期**：2026-05-04  
**状态**：已确认  
**版本**：v49

---

## 1. 背景与目标

当前 `/stocks` 页面使用原始的 20 列白底表格，存在以下问题：
- 列数过多（20列）信息过载
- 无视觉层次、涨跌无颜色区分
- 移动端完全不可用
- 分页固定定位遮挡内容

目标：改造为**现代卡片风格**，深色主题，精选核心字段，红涨绿跌直觉化展示。

## 2. 设计决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 布局风格 | 现代卡片风（方案B） | 用户选择，视觉清晰，关键数据突出 |
| 配色主题 | 深色主题 | 用户选择，护眼，专业券商feel |
| 展示字段 | 8个核心字段 | 代码、名称、最新价、涨跌幅、涨跌额、成交量、换手率、市盈率 |
| 涨跌色彩 | 红涨绿跌 | A股惯例 |
| 版本号 | 保留 `v{count} \| {commitId}` | 已有功能不变 |

## 4. 卡片布局明细

每张卡片包含：
- **左侧边条**：4px 宽色条（红=涨，绿=跌，灰=平）
- **顶行**：代码(粗体) | 名称 | 市场标签(沪/深) | 交易时间
- **核心行**：大号最新价 | 涨跌幅色块标签 | 涨跌额
- **辅助行**：成交易图标 成交量 | 换手率 | 市盈率

## 5. CSS 设计规范

### 5.1 颜色系统

| 用途 | 颜色 | 色值 |
|------|------|------|
| 页面背景 | 深蓝灰 | `#0f0f1a` |
| 卡片背景 | 深灰蓝 | `#1a1a2e` |
| 上涨色 | 红 | `#e94560` |
| 下跌色 | 绿 | `#4caf50` |
| 文本主色 | 亮白灰 | `#e0e0e0` |
| 文本次色 | 中灰 | `#888` |
| 卡片边框 | 极深蓝 | `#16213e` |

### 5.2 字体大小

| 元素 | 大小 | 粗细 |
|------|------|------|
| 代码 | 14px | 600 |
| 名称 | 13px | 400 |
| 最新价 | 24px | 700 |
| 涨跌幅标签 | 14px | 600 |
| 辅助字段 | 12px | 400 |

### 5.3 响应式断点

- **PC (≥1024px)**：卡片最大宽度 800px，居中
- **平板 (768-1023px)**：卡片全宽
- **手机 (<768px)**：卡片全宽，字略缩小

## 6. 数据流

```
                                          ┌──────────────────┐
                                          │  list.html       │
  ┌─────────┐    ┌──────────────────┐     │  (Thymeleaf)     │
  │ MySQL   │───▶│ StockRepository   │────▶│  stocks.content  │
  │ stock   │    │ findAll(pageable) │     │     ↓            │
  └─────────┘    └──────────────────┘     │  th:each 遍历     │
                                          │  渲染卡片         │
  ┌──────────┐   ┌──────────────────┐     │  版本号 footer    │
  │ git.prop │──▶│ VersionConfig    │─────│                  │
  └──────────┘   └──────────────────┘     └──────────────────┘
```

**无需改动后端代码**（Controller/Service/Repository 不变），仅改 HTML 模板和 CSS。

## 8. 降级与兼容

- **无 JS 依赖**：纯 CSS + Thymeleaf，不依赖 JavaScript 框架
- **旧浏览器兼容**：使用标准 CSS3，IE11 不保证
- **数据缺失**：字段 null 时显示 `--`
